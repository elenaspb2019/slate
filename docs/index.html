    Благотворительность

    Описание

    Вместе с оплатой заказа пользователь может перечислить деньги в один из благотворительных фондов. Работает только для платежей при выборе способа оплаты на стороне Яндекс.Кассы.

    Ниже представлен запрос на платеж с благотворительным взносом в фонд Деда Мороза «Волшебная ёлка». Фонд поддерживает питомники для новогодних ёлок: высаживает ёлки, чтобы не снижалась численность деревьев.

    Пользователь во время выбора способа платежа через Яндекс.Кассу может сделать пожертвование в фонд Деда Мороза «Волшебная ёлка», поставив галочку напротив предложения и выбрав соответствующие параметры. Тем самым сумма платежа будет пересчитана с учетом пожертвования, которое будет отправлено в фонд Деда Мороза «Волшебная ёлка».

    Чтобы посмотреть, как работают другие благотворительные программы Яндекса, перейдите по ссылке или прочтите статью из блога Яндекс.Кассы.

    Быстрый старт

    Мы предполагаем, что если вы здесь, значит, вы зарегистрировались в Яндекс.Кассе, получили доступ к личному кабинету, сгенерировали и активировали секретный ключ.

    Что дальше:

    1. Создайте платеж и получите ссылку для оплаты
    2. Перенаправьте пользователя на страницу оплаты
    3. Дождитесь успешного завершения платежа

    Шаг 1. Создайте платеж

    <Идентификатор магазина> — shopId в личном кабинете.

    <Секретный ключ> выпустите по инструкции.

    В качестве <Ключа идемпотентности> используйте любое значение, уникальное для этого запроса. Мы рекомендуем v4 UUID.

    curl https://payment.yandex.net/api/v3/payments \
  -X POST \
  -u <Идентификатор магазина>:<Секретный ключ> \
  -H 'Idempotence-Key: <Ключ идемпотентности>' \
  -H 'Content-Type: application/json' \
  -d '{
        "amount": {
          "value": "2.00",
          "currency": "RUB"
        },
        "confirmation": {
          "type": "redirect",
          "return_url": "https://www.merchant-website.com/return_url"
        },
        "capture": true,
        "description": "Заказ №1",
        "offer_charity": false
      }'
"

    Создайте платеж с выбором способа оплаты на стороне Яндекс.Кассы. Для этого отправьте Яндекс.Кассе запрос на создание платежа:

    • в объекте amount передайте сумму и валюту платежа;
    • в объекте confirmation передайте тип redirect и адрес страницы на вашей стороне, на которую пользователь вернется после оплаты (в параметре return_url);
    • в параметре capture передайте значение true, чтобы платеж автоматически перешел в статус succeded после оплаты;
    • в параметре description передайте комментарий к платежу (он отобразится в личном кабинете);
    • в параметре offer_charity передайте значение true, если хотите, чтобы на странице оплаты отобразилась дополнительная платежная форма одной из благотворительных акций Яндекс.

    Запрос отправьте с уникальным ключом идемпотентности.

    Как только пользователь выберет способ и подтвердит оплату, платеж перейдет в статус succeeded. Этот статус означает, что платеж успешно завершен и вы можете выдать заказ.

    Вы можете узнать об изменении статуса платежа двумя способами:

    • дождаться уведомления от Яндекс.Кассы (оно придет на URL, который вы укажете в настройках магазина в личном кабинете);
    • запрашивать информацию о платеже с удобной для вас периодичностью (ключ идемпотентности не нужен).

    Использование API

    Платежи

    API позволяет создавать, подтверждать, отменять платежи, а также получать информацию о них. Как провести платеж

    Объект платежа

    Пример объекта Payment

    Объект платежа (Payment) содержит всю информацию о платеже, актуальную на текущий момент времени. Он формируется при создании платежа и приходит в ответ на любой запрос, связанный с платежами.

    Параметры
    charity object
    Сопутствующие денежные перечисления на благотворительность
    type object required
    Название благотворительной организации
    donator object
    Имя пользователя, который сделал взнос (для публикации на сайте фонда)
    donation object required
    Данные о пожертвовании
    type object required
    Тип объекта donation:
    money – деньги, произвольная сумма
    seeding – саженцы
    tree – деревья
    quantity object
    Положительное целое число. Используется для расчета amount
    amount object required
    Размер donation. Для type seedling и tree рассчитывается на основе quantity
    value string required
    Сумма в выбранной валюте. Выражается в виде строки и пишется через точку, например 10.00. Количество знаков после точки зависит от выбранной валюты
    currency string required
    Код валюты в формате ISO-4217. Должен соответствовать валюте субаккаунта (recipient.gateway_id), если вы разделяете потоки платежей, и валюте аккаунта (shopId в личном кабинете), если не разделяете

    Чтобы принять оплату, необходимо создать объект платежа — Payment. Он содержит всю необходимую информацию для проведения оплаты (сумму, валюту и статус). У платежа линейный жизненный цикл, он последовательно переходит из статуса в статус.

    В ответ на запрос придет объект платежа в актуальном статусе.

    Параметры запроса
